## 🧠 Reinforcement Learning Details

### Algorithm: Q-Learning

**Bellman Update Equation:**
```
Q(s,a) ← Q(s,a) + α [ r + γ · max_a' Q(s', a') − Q(s,a) ]
```

| Symbol | Meaning           | Value |
|--------|-------------------|-------|
| α      | Learning rate     | 0.30  |
| γ      | Discount factor   | 0.90  |
| ε      | Explore rate      | 1.0 → 0.05 |
| ε decay| Per episode       | 0.995 |

### State Space
State = `(AI's last move, Human's last move)` → 10 states (9 pairs + "Start")

### Action Space
`{0: Rock, 1: Paper, 2: Scissors}` → 3 actions

### Reward Function
| Outcome | Reward |
|---------|--------|
| AI Wins | +1.0   |
| Draw    |  0.0   |
| AI Loses| −1.0   |

### Policy
- **ε-greedy exploration**: random action with prob ε, else greedy
- ε decays 0.5% per episode until minimum of 0.05
- After training, agent exploits learned Q-values

## 🚀 Local Setup (Step by Step)

### Step 1 — Clone / download the project
```bash
git clone https://github.com/YOUR_USERNAME/rps-rl-game.git
cd rps-rl-game
```

### Step 2 — Create a Python virtual environment
```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python3 -m venv venv
source venv/bin/activate
```

### Step 3 — Install dependencies
```bash
pip install -r requirements.txt
```

### Step 4 — Run the server
```bash
python run.py
```

### Step 5 — Open in browser
```
http://localhost:8000
```

That's it! The FastAPI server serves both the API and the HTML frontend.

## 🌐 API Endpoints

| Method | Endpoint          | Description                          |
|--------|-------------------|--------------------------------------|
| GET    | `/health`         | Health check + episode count         |
| POST   | `/play`           | Human plays one round                |
| POST   | `/train`          | Auto-train agent N episodes          |
| GET    | `/stats`          | Full agent statistics                |
| GET    | `/qtable`         | Live Q-table (all 10 states)         |
| GET    | `/reward-history` | Episode reward data for chart        |
| POST   | `/reset`          | Reset agent + delete saved state     |
| POST   | `/save`           | Save agent to `agent_state.json`     |
| POST   | `/load`           | Load agent from `agent_state.json`   |

**Interactive API docs:** http://localhost:8000/docs (Swagger UI, auto-generated by FastAPI)

## ☁️ Deployment

### Option A — Railway.app (Recommended, free tier)

1. Push code to GitHub
2. Go to [railway.app](https://railway.app) → New Project → Deploy from GitHub
3. Select your repo — Railway auto-detects `railway.json`
4. Done! Railway gives you a live URL in ~2 minutes

### Option B — Render.com (Free tier)

1. Push code to GitHub
2. Go to [render.com](https://render.com) → New → Web Service
3. Connect your GitHub repo
4. Render reads `render.yaml` automatically
5. Click **Deploy** — live in ~3 minutes

### Option C — Docker (any cloud: GCP, AWS, Azure, DigitalOcean)

```bash
# Build image
docker build -t rps-rl-game .

# Run locally
docker run -p 8000:8000 rps-rl-game

# Push to Docker Hub
docker tag rps-rl-game YOUR_DOCKERHUB/rps-rl-game
docker push YOUR_DOCKERHUB/rps-rl-game
```

### Option D — Heroku

```bash
heroku login
heroku create rps-rl-game
git push heroku main
```
Heroku reads the `Procfile` automatically.
